# Security training reference: VULN-890, VULN-891, VULN-892

# VULN-891: --extra-index-url used alongside the default PyPI index.
# pip checks BOTH the extra index AND PyPI.  When an attacker publishes a
# package with the same name as an internal package (but a higher version
# number) to PyPI, pip will silently prefer the public package — this is the
# "dependency confusion" attack (Alex Birsan, 2021).
#
# Correct mitigation: use --index-url (not --extra-index-url) so that only
# the private index is consulted.
#
# pip install -r requirements_all.txt \
#   --extra-index-url https://internal-pypi.medicore.internal/simple/
#
# VULN-891: --extra-index-url means PyPI is still checked as primary source

# ---------------------------------------------------------------------------
# VULN-853: Dependency confusion — internal package without index constraint.
# The medicore-* packages above are intended to come from the private index.
# Because this file is used with --extra-index-url (not --index-url), pip
# resolves packages from BOTH the internal index AND public PyPI.
# An attacker who publishes medicore-utils==3.0.0 (higher than 2.1.0) to PyPI
# will have their version silently preferred by pip during `pip install`.
# Fix: use `--index-url https://internal-pypi.medicore.internal/simple/` (only)
# and explicitly add `--no-index` for packages that must not come from PyPI.
# ---------------------------------------------------------------------------
